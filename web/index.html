PyDevices / pydisplay_android

Native Android APK for the pydisplay stack.

pydisplay_android is the native APK companion to pydisplay: python-for-android recipes, a buildozer-driven demo under p4a_app/, and the smoke path that keeps the mobile build honest. On Android there is no MicroPython port — the app runs CPython with the SDL2 bootstrap, pulling wheels from TestPyPI.

This is not the browser PWA. The installable PyScript gallery lives on the pydisplay Pages site; this repo produces a real APK you install on a phone or emulator.

Build the demo APK

requires Android SDK/NDK + buildozer 
git clone https://github.com/PyDevices/pydisplay_android.git
cd pydisplay_android

./build_android.sh              # prompts for launcher title
./build_android.sh -y           # keep current title (CI)
./build_android.sh --title Paint

# APK under p4a_app/bin/
./scripts/emulator.sh

Runtime / SDL (from p4a_app)

same Runtime idiom as desktop pydisplay

board_config

SDLDisplay + eventsys.Runtime — from p4a_app/board_config.py.

from displaysys.sdldisplay import SDLDisplay as DTDisplay, get_events
import eventsys

display_drv = DTDisplay(
    width=720, height=1280,
    rotation=0, title=..., scale=1.0,
    window_flags=...,
)
runtime = eventsys.Runtime(display=display_drv, host_read=get_events)

paint handlers

From p4a_app/paint.py — register events, then run forever.

from board_config import display_drv, runtime

runtime.on(runtime.events.MOUSEBUTTONDOWN, handler)
runtime.on(runtime.events.MOUSEMOTION, on_motion)
runtime.run_forever()

PWA vs APK

two different delivery paths — do not conflate
PWA

Browser / PyScript

Runs in Chrome (or as an installed web app). MicroPython or Pyodide in the browser — no APK, no Android SDK. Ideal for demos and quick tryouts.

Open pydisplay PWA
APK

Native Android

This repo. CPython inside a python-for-android APK with SDL2, usdl2, and TestPyPI wheels. Install with adb or the emulator script — not via the browser gallery.

Customize

clone and replace the demo app

p4a_app/

Demo application: board_config.py, paint.py, main.py, and buildozer.spec (package id org.pydevices.p4a_app).

p4a_recipes/

Recipes for usdl2, graphics, displaysys, eventsys, multimer, and optional lvglcpython — versions left unpinned so pip takes the latest matching TestPyPI wheel.
